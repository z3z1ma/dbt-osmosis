# Mission

Fix `ticket:c10lint24` user-facing lint/diff CLI behavior without rewriting the SQL linter engine or widening public command contracts beyond the ticket acceptance criteria.

# Bound Context

`ticket:c10lint24` advances `initiative:dbt-110-111-hardening#OBJ-006`: user-facing CLI surfaces should fail clearly, respect documented selectors/options, and have coverage for advertised behavior.

The ticket is bounded to these claims:

- `ticket:c10lint24#ACC-001`: `dbt-osmosis diff schema my_model` scopes to `my_model` or exits clearly if positional selectors are unsupported.
- `ticket:c10lint24#ACC-002`: unknown selectors do not silently run against the whole project.
- `ticket:c10lint24#ACC-003`: `lint file ... --disable-rules select-star` suppresses the disabled rule.
- `ticket:c10lint24#ACC-004`: `--rules` and `--disable-rules` precedence is deterministic and documented in tests/docs.
- `ticket:c10lint24#ACC-005`: CLI output does not duplicate errors/warnings under `Other`.
- `ticket:c10lint24#ACC-006`: lint project/model selection matches documented package/ephemeral inclusion behavior.

# Source Snapshot

Relevant current observations from parent inspection at source fingerprint `b2e4e9825b5b0e6de372c8ae9cefd5b6e2d6e926`:

- `src/dbt_osmosis/cli/main.py:1637-1756` defines `diff schema` with `@yaml_opts`, so it accepts positional `models`; the command builds `YamlRefactorSettings` without passing `models` or `fqn`, then manually filters only when `fqn` is set. Positional selectors are currently ignored.
- `SchemaDiff.compare_all()` already delegates to `_iter_candidate_nodes(self._context)` when `nodes` is `None`, so passing selector settings into the `YamlRefactorContext` should keep diff selection aligned with shared node filters.
- `src/dbt_osmosis/cli/main.py:2201-2295` accepts `lint file --disable-rules` but calls `lint_sql_code()` with only `rules=enabled_rules`.
- `src/dbt_osmosis/core/sql_lint.py:693-717` has `lint_sql_code(context, raw_sql, dialect=None, rules=None)` and constructs `SQLLinter(dialect=dialect, enabled_rules=rules)` without disabled rules.
- `SQLLinter._filter_rules()` currently returns early for `enabled_rules`, so if both enabled and disabled rule lists are supplied the disabled list cannot apply. Define the intended precedence explicitly in tests. The recommended contract for this ticket is: start from `--rules` when provided, then remove anything named by `--disable-rules`; disabled rules win on overlap.
- `src/dbt_osmosis/cli/main.py:2267-2290`, `2365-2387`, and `2493-2496` build `other` by comparing `LintViolation.level` enum values to strings. Because `LintViolation.level` is `LintLevel`, errors/warnings can be reprinted under `Other`.
- `src/dbt_osmosis/core/sql_lint.py:603-618` has custom lint node iteration: models only, no external package filter, no ephemeral skip, and substring FQN matching. Shared YAML selection in `node_filters.py` excludes external packages unless `include_external`, excludes ephemeral models, and uses `_is_fqn_match()` for partial FQN segment matching.
- Existing SQL lint tests live in `tests/core/test_sql_lint.py`; existing high-level CLI tests live in `tests/core/test_cli.py`; schema diff tests live in `tests/core/test_diff.py`.
