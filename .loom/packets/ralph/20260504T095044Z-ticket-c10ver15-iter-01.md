# Mission

Versioned model YAML can preserve or lose user-authored descriptions, meta, tags, tests, and validation feedback, so the implementation must be targeted and tested against versioned model fixtures.

Use local validation only for this ticket. Do not wait on GitHub Actions; final initiative validation will handle CI later.

# Source Snapshot

Current state:

- `src/dbt_osmosis/core/inheritance.py::_get_node_yaml()` returns the top-level model YAML entry for `ModelNode` and `SeedNode`; it does not select `models[].versions[]` for versioned models.
- `src/dbt_osmosis/core/introspection.py::PropertyAccessor._get_from_yaml()` reads columns from the mapping returned by `_get_node_yaml()`, so version-level columns are invisible when `_get_node_yaml()` returns the top-level model entry.
- `src/dbt_osmosis/core/schema/validation.py::ModelValidator._validate()` validates top-level model tests and `model.get("columns", [])`, but does not validate `versions`, `versions[].v`, or `versions[].columns`.
- `demo_duckdb/models/staging/jaffle_shop/main/stg_customers.yml` already has `models[].versions[]` entries for `stg_customers` v1 and v2 with version-level columns.
- `sync_operations.py` already has versioned sync helpers (`_get_or_create_version`, `_sync_versioned_model`), so avoid redesigning sync unless tests prove a direct gap in this ticket's scope.
