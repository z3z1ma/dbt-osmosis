# Mission

Ensure dry-run/check-mode YAML sync and write paths cannot leave dirty global YAML buffers that affect later reads or writes in the same process, and make tests reset production cache instances rather than replacing them with plain dicts.

# Bound Context

`ticket:c10dry17` is medium-risk `code-behavior` work under `initiative:dbt-110-111-hardening#OBJ-003` and `#OBJ-004`. Dry-run and `--check` are safety-critical surfaces, and cache replacement in tests can hide production cache bugs.

Use local validation only for this ticket. Do not wait on GitHub Actions; final initiative validation will handle CI later.

# Source Snapshot

Current state at packet compilation:

- `src/dbt_osmosis/core/schema/writer.py::_write_yaml()` detects dry-run changes and tracks mutations, but dry-run branches do not discard dirty YAML cache state.
- `src/dbt_osmosis/core/schema/writer.py::commit_yamls()` similarly detects dry-run changes for buffered writes, but dry-run branches leave buffer/original caches populated.
- `src/dbt_osmosis/core/sync_operations.py::_finalize_synced_document()` marks YAML caches dirty before checking `commit`; `commit=False` returns without clearing or restoring disk-backed cache state.
- `src/dbt_osmosis/core/schema/reader.py` owns `_YAML_BUFFER_CACHE`, `_YAML_ORIGINAL_CACHE`, `_YAML_BUFFER_CACHE_LOCK`, `_mark_yaml_caches_dirty()`, and `_discard_yaml_caches()`.
- `tests/conftest.py::fresh_caches` currently monkeypatches `_COLUMN_LIST_CACHE` and `_YAML_BUFFER_CACHE` to `{}`, bypassing production cache class behavior and not resetting `_YAML_ORIGINAL_CACHE`.
