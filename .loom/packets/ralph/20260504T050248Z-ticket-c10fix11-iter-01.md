# Mission

Make the dbt fixture and integration smoke paths hermetic enough that tests do not depend on source-tree `demo_duckdb/target`, repo-root DuckDB files, stale manifests, or destructive fixture cleanup.

# Bound Context

`ticket:c10fix11` is high-risk `code-behavior` work under `initiative:dbt-110-111-hardening#OBJ-004`. Fixture isolation underpins the reliability of all dbt 1.10/1.11 compatibility claims, so this iteration must be test-first and must not run the current destructive `demo_duckdb/integration_tests.sh` shape before making it safe.

Stay within fixture/test/integration-smoke boundaries. Do not change demo project semantics except where a temp-copy rewrite is required for isolation. Do not use `git checkout`, `git clean`, `git reset`, or other destructive cleanup as normal test behavior.

# Source Snapshot

Current implementation observations:

- `demo_duckdb/profiles.yml` has `test.path: "test.db"`, so dbt commands can resolve the DuckDB file relative to the current working directory instead of the copied project directory.
- `tests/conftest.py` copies `demo_duckdb` to a temp template and excludes `target/` only when `exclude_target=True`. It does not exclude ignored DuckDB files, logs, `dbt_packages`, or other generated artifacts by default.
- `tests/conftest.py` changes cwd to the copied template before running dbt commands because the profile path is relative. Later `yaml_context` creates dbt context from copied paths without changing cwd.
- `tests/core/conftest.py` autogenerates or reuses `demo_duckdb/target/manifest.json` in the source tree and refreshes only by source file mtimes, not by dbt/core/adapter/target identity.
- `tests/core/test_legacy.py` reads `Path("demo_duckdb/target/manifest.json")` directly.
- `demo_duckdb/integration_tests.sh` changes into the source fixture, runs `uv run dbt-osmosis ...`, mutates tracked YAML, then restores with `git checkout -- models/ seeds/` and `git clean -fd models/ seeds/`.
- `.github/workflows/tests.yml` currently runs the integration smoke inline from `working-directory: demo_duckdb`, mutates source checkout files, and restores with `git checkout` / `git clean`.
- `Taskfile.yml` matrix runs `dbt parse --project-dir demo_duckdb --profiles-dir demo_duckdb -t test` and `pytest`; keep local workflow coherent if helper APIs or script usage change.
- `wiki:ci-compatibility-matrix` currently warns that `demo_duckdb/integration_tests.sh` remains unsafe to reintroduce into matrix CI unchanged.
