# Source Snapshot

Relevant current behavior:

- `SettingsResolver.resolve()` uses `column.meta`, `column.meta[dbt-osmosis-options]`, and `column.meta[dbt_osmosis_options]` for column-level settings, but not `column.config.meta`.
- `PropertyAccessor._get_from_manifest()` returns `column.meta` and `column.tags` for column-level properties, but not `column.config.meta` or `column.config.tags`.
- Inheritance graph creation has partial config cleanup/merge behavior, but inherited metadata exposed to `inherit_upstream_column_knowledge()` can miss metadata when it exists only under `config.meta` or `config.tags`.
- dbt docs/source research says dbt 1.10+ supports column `config.meta` and `config.tags` while legacy top-level fields still exist.
