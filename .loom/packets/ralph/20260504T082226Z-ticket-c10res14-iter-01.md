# Source Snapshot

Current state:

- `src/dbt_osmosis/core/introspection.py::SettingsResolver.resolve()` is mostly node-only. It checks column meta, node meta, `config.extra`, `config.meta`, and `unrendered_config`, but it does not use context-backed `SupplementaryFileSource` or `ProjectVarsSource`.
- `ProjectVarsSource` and `SupplementaryFileSource` already exist but are not part of normal resolver behavior.
- `_get_setting_for_node()` delegates to the stateless singleton resolver and cannot see project vars or supplementary files.

- `tests/core/test_real_config_shapes.py` already proves real dbt 1.10+ parsed shapes for `config.meta`, `config.extra`, `unrendered_config`, and column `config.meta`/`config.tags`.
