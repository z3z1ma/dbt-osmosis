# Task For This Iteration

1. Add failing tests first for the feed hardening behavior. Cover at least:
   - default/disabled feed path does not call outbound network code;
   - enabled feed failure, including parser failure, returns safe fallback HTML and does not raise;
   - malicious title/link/comments/published fields are escaped or rejected so raw `<script>`/HTML and `javascript:` URLs are not injected;
   - CLI help or launch wiring exposes an explicit `--enable-external-feed` opt-in and passes it to the Streamlit app.
2. Implement the smallest safe behavior:
   - default external feed disabled;
   - `dbt-osmosis workbench --enable-external-feed` passes `--enable-external-feed` to the Streamlit app;
   - app parser accepts `--enable-external-feed`;
   - when disabled, startup does not make an outbound RSS request and renders a safe static message;
   - when enabled, fetch with a short timeout using stdlib URL retrieval or an equivalent timeout-bound path;
   - catch network and feedparser/parser failures and render a safe fallback message;
   - escape text and attribute values and reject non-HTTP(S) feed URLs before sending strings to `InnerHTML`.
3. Update CLI reference docs only if a new CLI option is added.
4. Keep optional dependency behavior clear; do not remove workbench extras or add sanitization dependencies unless unavoidable.
