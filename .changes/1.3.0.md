## 1.3.0 - 2026-03-24
### Added
* Add external YAML formatter integration via --formatter flag (#323)

- New --formatter CLI option on refactor, organize, and document commands
- Runs any external formatter (prettier, yamlfmt, yq, etc.) on written files after completion
- Failure is non-fatal: logs a warning but does not fail the command
- Simplifies CI pipelines from 3-step (osmosis → formatter → diff) to 2-step
- Example: dbt-osmosis yaml refactor --formatter "prettier --write"
* Add fusion_compat mode for dbt 1.9.6+ and dbt Fusion YAML output (#317)

- Nests column-level meta and tags inside config: blocks as required by dbt >= 1.9.6
- Auto-detects from Fusion manifest in target/, Fusion binary on PATH, or dbt-core version >= 1.9.6
- CLI flags: --fusion-compat / --no-fusion-compat on yaml refactor, yaml organize, and yaml document
- Fully backwards compatible: dbt < 1.9.6 users see no change (classic format preserved)
- Addresses #301 (dbt 1.11 meta duplication) and #278 (empty config blocks)
* Add Python 3.13 support and upgrade ydata-profiling (#318)

- Upgrade ydata-profiling from ~=4.13.0 to >=4.17.0 (removes htmlmin incompatibility)
- Extend requires-python to >=3.10,<3.14
- Add Python 3.13 to CI test matrix (dbt-core 1.10+ only, as 1.8/1.9 are unsupported on 3.13)
