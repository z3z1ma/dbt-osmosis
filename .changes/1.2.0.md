## 1.2.0 - 2026-01-03
### Added
* Add dbt 1.10+ compatibility for meta/tags namespace change
* Add --output-to-upper flag for Snowflake users to control case sensitivity in column names
* Add column-level and model-level progenitor overrides for documentation inheritance control
* Add --include-external CLI option to include dbt packages in YAML processing
* Add multi-database integration tests support (Postgres) for improved cross-database compatibility
* Add LRU cache implementation to prevent memory leaks in YAML buffer cache
* Add dbt-core-interface integration to reduce database coupling
* Add comprehensive plugin system with fuzzy matching capabilities (FuzzyCaseMatching, FuzzyPrefixMatching)
* Add comprehensive test coverage including security tests and edge cases
* Add YAML reorganization behavior tests for source_name template handling
* Add support for Google Gemini and Anthropic models as LLM providers for documentation synthesis
* Add support for BigQuery column descriptions in YAML documentation
* Add support for node.config.meta in YAML path template retrieval
* Add --log-level CLI option for all commands to control logging verbosity
* Add tutorial for YAML synthesis using AI/LLM providers
* Unified configuration resolution system with SettingsResolver class supporting 7-source precedence chain (column meta, node meta, config.extra, config.meta, unrendered_config, project vars, dbt-osmosis.yml)
* PropertyAccessor class for unified model property access from manifest (rendered) or YAML (unrendered jinja) sources
* Support for dbt-osmosis.yml supplementary configuration file for project-level defaults outside dbt's hot path
* Column-level configuration override support allowing individual columns to override node-level dbt-osmosis settings
* Add `prefer-yaml-values` setting and `--prefer-yaml-values` CLI flag to preserve unrendered jinja templates ({{ var() }}, {{ env_var() }}, etc.) in YAML fields. This addresses issue #266 where `{{ var() }}` in `policy_tags` was being replaced with rendered values. Additionally, expanded `PropertyAccessor._has_unrendered_jinja()` to detect all jinja patterns including {{ var() }}, {{ env_var() }}, and generic {{ }} expressions, with support for nested list and dict structures.
* Add --prefer-yaml-values flag to preserve unrendered jinja templates in YAML fields. Fixes #266
### Changed
* Loosen version constraints on dbt-core
* Drop support for Python 3.9 (minimum is now Python 3.10)
* Optimize test fixtures with session-level dbt build
* Extract helper functions from _build_column_knowledge_graph for better modularity
* Refactor osmosis.py into modular core modules (config, inheritance, introspection, path_management, restructuring, schema, transforms, sync_operations)
* path_management.py now uses SettingsResolver.get_yaml_path_template() for consistent config resolution
* _get_setting_for_node() now delegates to SettingsResolver internally, maintaining full backward compatibility with all 24 existing call sites
* Upgrade dbt-core to 1.11.2 and remove upper version constraints on dbt dependencies
### Fixed
* Fix target directory exclusion when copying demo project for session templates
* Fix empty_yaml_template security test mock
* Fix dbt 1.10 compat layer that broke osmosis_progenitor tracking
* Fix preservation of unrendered descriptions when use_unrendered_descriptions is True
* Fix KeyError: 'duckdb' and adapter.connections.close() issues
* Fix catalog data types usage when --catalog-path is provided
* Fix dict-like methods (keys, values, items) on LRUCache class
* Fix test fixtures to use conftest.py yaml_context
* Fix test failures in column introspection and inheritance
* Fix removal of empty config keys from injected columns
* Fix YAML inheritance test isolation with function-scoped fixtures
* Fix sql_operations tests for dbt-core-interface
* Fix threading locks to protect global caches from race conditions
* Fix test compatibility with dbt-core 1.10.x
* Fix data type handling to only skip when flag is specified
* Fix numeric-precision-and-scale and string-length options in node settings
### Security
* Fix critical security vulnerabilities
